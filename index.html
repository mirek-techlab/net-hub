CISCO ASA IPsec L2L IKEv2
Changing Peer IP — Site-to-Site VPN
ASA — IKEv2 / crypto map / tunnel-group migration
What to change
Commands
Verify
Must change
  • crypto map — set peer <NEW_IP>
  • tunnel-group <NEW_IP> — recreate
  • no tunnel-group <OLD_IP>
  • clear crypto ikev2 sa peer <OLD_IP>
Leave untouched
  • ikev2 policy (global)
  • ikev2 ipsec-proposal (by name)
  • crypto map ACL (match address)
  • PFS / SA lifetime settings
  • crypto map interface binding
⚠ GOTCHA — tunnel-group is keyed by peer IP
On ASA, L2L tunnel-group name IS the peer IP. You cannot rename it — must delete and recreate. Pre-shared keys (type 8) can be pasted as-is from running config.
✓ FAILOVER — always configure on active unit
Changes on active sync to standby automatically. Never configure directly on standby — breaks sync immediately.
Pre-change audit
# Find all references to old peer IP
show run crypto map
show run tunnel-group 169.51.163.157
show run | i 169.51.163.157
Change sequence
# 1. Update crypto map peer
crypto map IA_map1 11 set peer <NEW_IP>

# 2. Create new tunnel-group
tunnel-group <NEW_IP> type ipsec-l2l
tunnel-group <NEW_IP> general-attributes
 default-group-policy GroupPolicy11
tunnel-group <NEW_IP> ipsec-attributes
 ikev2 remote-authentication pre-shared-key 8 <key>
 ikev2 local-authentication pre-shared-key 8  <key>

# 3. Remove old tunnel-group
no tunnel-group 169.51.163.157

# 4. Clear old SA
clear crypto ikev2 sa peer 169.51.163.157
Verify tunnel is up
show crypto ikev2 sa
# Expect: UP-ACTIVE on new peer

show crypto ipsec sa peer <NEW_IP>
# Check: encaps/decaps counters incrementing

show crypto ikev2 sa detail
# Check: Status = READY, correct local/remote addr
Troubleshooting — IKEv2 tunnel not coming up
1
Check SA state
show crypto ikev2 sa  # MM_WAIT / negotiating / empty?
2
Check reachability
ping <NEW_IP> source <outside-int>  # UDP/500 blocked?
3
Enable debug
debug crypto ikev2 protocol 5
debug crypto ikev2 platform 5
# Look for: proposal mismatch, PSK fail, ID mismatch
4
Common failures after peer IP change
tunnel-group missing or wrong IP
show run tunnel-group — confirm new IP, old is gone
crypto map still pointing old peer
show run crypto map — verify set peer = NEW_IP
ghost SA from old peer
clear crypto ikev2 sa peer <OLD_IP> + clear crypto ipsec sa peer <OLD_IP>
5
Always turn off debug when done
no debug all  # high CPU risk on production ASA
